# 个人实践心得（组员 C）

本阶段我主要负责测试模块与自动化测试链路的打通。最初在执行 `aa test` 时遇到 hvigor 配置错误、测试模块类型不匹配，以及 `OpenHarmonyTestRunner` 缺失导致的启动异常。经过逐项排查，我体会到测试体系的稳定性不是单一代码问题，而是“构建配置 + 模块声明 + 测试入口 + 运行环境”共同决定的结果。尤其是 `ohosTest` 模块的类型、`build-profile.json5` 的目标配置，以及测试运行器入口路径，任何一个细节不一致都会导致用例无法启动。

在解决问题的过程中，我对 Hypium 的运行机制有了更清晰的理解：测试并不是直接启动业务 Ability，而是由 TestRunner 解析参数并执行测试集合，最终把结果反馈给系统。为了更快定位问题，我增加了测试输出日志，这让我意识到“可观察性”是测试成功的关键。

后续我希望继续补齐 UI 手工测试与性能量化数据，形成更完整的测试闭环。也希望团队在真实网络接口接入后，能扩展更多异常场景与稳定性测试，让最终材料更贴近真实项目要求。
