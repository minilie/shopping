# 个人实践心得（组员 B）

本阶段我负责搜索与筛选体验、排序规则落地以及商品卡片的视觉优化。做完这部分后最大的体会是：功能“能用”并不等于体验“好用”。最初筛选区占用屏幕过大，影响商品浏览效率，因此我改为折叠式筛选面板，让搜索成为默认入口，把筛选作为次级操作，这样更符合常见电商应用的习惯。

在实现层面，我把搜索、排序、价格筛选统一收敛到 `ProductListDataSource` 中处理，避免 UI 直接拼接逻辑，确保分页加载与筛选结果一致。处理滚动与刷新时，我也意识到组件嵌套后“顶部判断”并不可靠，需要通过列表滚动状态来控制刷新触发，避免数据被误刷新。

后续我希望与组内同学一起完成真机测试与性能量化，并进一步优化数据来源和筛选维度，让项目更贴近真实业务场景。
